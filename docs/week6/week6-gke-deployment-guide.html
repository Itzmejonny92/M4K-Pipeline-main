Step-by-Step Guide

Setup Your Environment

Get your kubeconfig file from the Team Flags portal and configure kubectl:

# Save kubeconfig mkdir -p ~/.kube cp team-kubeconfig.yaml ~/.kube/config # Test connection kubectl get nodes
✅ You should see the GKE cluster nodes listed!

Customize Terraform Configuration

Edit terraform/main.tf with your team details:

variable "team_name" { default = "your-namespace" # e.g., "lillteamet" }
⚠️ Use your Kubernetes namespace name (lowercase, no spaces)

Deploy Infrastructure with Terraform

cd terraform # Initialize Terraform terraform init # Preview changes terraform plan # Apply configuration terraform apply # Type 'yes' when prompted

What this creates:

  • ✅ Namespace (your isolated environment)
  • ✅ Resource Quotas (5 pods, 2 CPU, 4GB RAM)
  • ✅ Service Account (for monitoring)
  • ✅ RBAC Roles & Bindings
  • ✅ ConfigMap & Secret

Deploy Your Application

Edit kubernetes/app-deployment.yaml with your namespace:

metadata: namespace: your-namespace # Update this!

Then deploy:

kubectl apply -f kubernetes/app-deployment.yaml kubectl apply -f kubernetes/monitor-deployment.yaml # Check status kubectl get pods -n your-namespace

Verify & Celebrate! 🎉

# Watch monitor logs kubectl logs -f deployment/team-monitor -n your-namespace

You should see:

✅ Status reported to Team Flags
Pods: 3/3 running | Deployments: 2/2 ready | URL: ✅ 200

Architecture Overview

Component Purpose Type

Deployment Manages pods, ensures desired state Kubernetes
Service Internal load balancer for pods Kubernetes
Ingress Routes external traffic to services Kubernetes

Troubleshooting

Pods Not Starting

# Check pod status kubectl describe pod POD_NAME -n your-namespace kubectl logs POD_NAME -n your-namespace

Ingress Not Working

# Check ingress details kubectl describe ingress demo-app -n your-namespace

Monitor Not Reporting

# Check monitor logs kubectl logs deployment/team-monitor -n your-namespace # Verify ConfigMap kubectl get cm monitor-config -n your-namespace -o yaml
💡 Pro Tip: Most issues are related to namespace naming. Make sure you're using the correct namespace (lowercase) everywhere!
✅

Success Criteria

Your deployment is successful when:

  • All pods are in Running status
